## Entry point

`src/main.js` bootstraps the application in a specific order:

1. Creates the Vue app instance.
2. Registers Pinia (state), Vue Router (navigation), and vue-toastification (UI toasts).
3. Calls `authStore.inicializarSesion()` to restore the user session from Supabase before the app mounts.

```javascript
const app = createApp(App)

app.use(createPinia())
app.use(router)
app.use(Toast, {
  position: POSITION.TOP_RIGHT,
  timeout: 3000,
  closeOnClick: true,
  pauseOnHover: true,
})

const authStore = useAuthStore()
authStore.setRouter(router)

authStore.inicializarSesion().finally(() => {
  app.mount('#app')
})
```

<Note>
The app does not mount until `inicializarSesion()` completes. This prevents the router's auth guard from running before the session is known.
</Note>

## Vite configuration

The project uses Vite with two plugins and a path alias:

```javascript
import { fileURLToPath, URL } from 'node:url'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import vueDevTools from 'vite-plugin-vue-devtools'

export default defineConfig({
  plugins: [
    vue(),
    vueDevTools(),
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    },
  },
})
```

The `@` alias maps to `src/`, so `import { supabase } from '@/lib/supabase'` resolves to `src/lib/supabase.js` from any file in the project.

## Composition API pattern

All components use the `<script setup>` syntax. Stores are accessed via their composable functions:

```vue
<script setup>
import { useAuthStore } from '@/stores/auth'
import { useCartStore } from '@/stores/cartStore'

const authStore = useAuthStore()
const cartStore = useCartStore()
</script>

<template>
  <p>Welcome, {{ authStore.perfil?.nombre }}</p>
  <p>Items in cart: {{ cartStore.totalItems }}</p>
</template>
```
