S-mart uses a role-based access control (RBAC) system built on top of Supabase. Every authenticated user is assigned exactly one role, stored in the `usuarios` table. The Vue Router reads the `requiresRoles` meta field on each route and blocks navigation for users who lack the required role.

There are four named roles plus a default guest state for unauthenticated visitors:

| Role | Internal value | Primary use |
|---|---|---|
| Customer | `cliente` | Browse and purchase products |
| Seller | `vendedor` | List and manage products, process orders |
| Cashier | `cajero` | Process in-store point-of-sale transactions |
| Administrator | `administrador` | Full system access |

## How roles are enforced

### Router guard

The global `beforeEach` guard in `src/router/index.js` checks `to.meta.requiresRoles` on every navigation. If the current user's role is not in the array, they are redirected — admins go to `/admin`, sellers go to `/vendedor`, and everyone else goes to `/`.

```javascript
// src/router/index.js
if (to.meta.requiresRoles && !to.meta.requiresRoles.includes(userRole)) {
  if (authStore.esAdmin) return next({ name: 'admin' })
  else if (authStore.esVendedor) return next({ name: 'vendedor' })
  else return next({ name: 'home' })
}
```

### RoleGuard component

For conditional UI rendering within a page, use the `RoleGuard` component. It reads the current role from the auth store and only renders its slot if the role matches.

```vue
<RoleGuard :roles="['administrador', 'vendedor']">
  <button>Manage Products</button>
</RoleGuard>
```

### useRole composable

For programmatic role checks inside `<script setup>` blocks, import `useRole` from `src/composables/useRole.js`.

```javascript
import { useRole } from '@/composables/useRole'

const { isAdmin, canSell, hasRole } = useRole()
```

## Role assignment

Roles are stored in the `rol` column of the `usuarios` table in Supabase. When a user registers for the first time, `crearPerfil()` in `src/stores/auth.js` automatically inserts a row with `rol: 'cliente'`.

<Note>
  Changing a user's role must be done directly in the Supabase `usuarios` table or via the admin panel at `/admin/usuarios`. There is no self-service role upgrade flow.
</Note>
